# 需求工厂平台需求文档

## 1. 项目概述
构建一个面向内部运营与需求管理的统一平台——"需求工厂"。平台提供需求收集、文档生成、多格式产出（PPT、原型）、需求管理、开发协作等全流程功能。运营人员可通过首页推广案例，用户通过AI引导完成需求收集，最终产出标准化需求文档，并可与开发团队协同交付。

---

## 2. 用户角色与权限

| 角色 | 权限描述 |
|------|----------|
| **普通用户**（已注册登录） | 可创建自己的需求、查看和管理自己的需求列表；可编辑、提交开发、下载产出物；可查看个人资料。 |
| **平台管理员** | 可查看所有用户的需求；可管理用户（增删改查）；可配置首页案例。 |
| **交付团队**（特殊角色或外部系统） | 可接收"已提交开发"的需求，更新需求状态、上传代码包和部署手册。 |

## 3. 功能模块详细说明

### 3.1 首页
- **页面组成**：
  - 顶部导航栏：Logo、精选案例入口、登录/个人中心入口
  - **Hero区域**：左侧展示产品标题和描述，右侧为视觉图形，包含"立即开始体验"和"查看案例"按钮
    - 点击"立即开始体验" → 跳转到 AI引导需求收集页
  - **Features区域**：3个核心功能卡片展示（AI智能引导、自动化生成、快速交付）
  - **流程展示**：4步流程展示（提交想法 → AI澄清 → 生成文档 → 交付成果）
  - **案例展示区**：展示精选案例卡片（来源于后台管理推荐的案例）
    - 每个卡片包含：案例标题、简介、标签、缩略图
    - 点击卡片 → 进入案例详情页（可查看完整案例文档、原型、PPT，仅供预览复用）
  - **客户声音**：客户评价轮播展示
  - **CTA区域**：底部行动号召区域
  - **Footer**：页脚导航和版权信息

### 3.2 用户登录与个人中心
#### 3.2.1 登录方式
**集成公司Ops帐号体系**：
- 用户通过公司统一认证系统登录
- 支持SSO单点登录
- 登录后获取用户信息（姓名、部门、角色等）
- 与Ops系统用户体系打通，无需单独注册

#### 3.2.2 个人中心
- 右上角头像下拉菜单：
  - **我的需求**（进入需求管理）
  - **后台管理**（管理员可见）
  - **退出登录**

### 3.3 精选案例
- **案例列表页**：
  - 网格/列表双视图切换
  - 支持按行业、规模、标签筛选
  - 支持搜索案例名称和描述
  - 展示已发布的案例
- **案例详情页**：
  - 展示案例完整信息
  - 提供"复用此需求"按钮，可将案例内容快速填入AI引导流程

### 3.4 AI引导需求收集与文档生成（集成OpenClaw）

#### 3.4.1 页面布局
页面布局根据需求收集状态动态变化：

**状态一：初始收集阶段（需求文档未生成）**
- 页面为单列布局，居中显示**对话区**
- 顶部显示AI助手头像、名称和描述
- 展示快速分类入口：门户、低代码应用、流程平台、协同应用、智能体、搜问做、知识管理
- 用户可通过点击分类或输入内容开始对话

**状态二：文档已生成后**
- 页面切换为**两列分屏**：
  - **左侧对话区**：宽度固定为400px（可交互），保留对话历史，用户可继续修改需求
  - **右侧预览区**：宽度占据剩余屏幕，用于展示需求文档、PPT预览、原型预览
- 右侧预览区顶部有标签页（需求文档、PPT、原型），支持切换

#### 3.4.2 会话流程（OpenClaw集成）
1. 用户选择快速分类或直接输入需求描述
2. **OpenClaw通过多轮对话逐步完善需求结构**
3. OpenClaw实时分析用户输入，主动提问补充边界条件与技术约束
4. OpenClaw根据预设模板收集需求字段，计算进度百分比
5. 当收集完成后，OpenClaw自动生成需求文档（Markdown/JSON），并**自动切换为两列分屏**，右侧预览区显示生成的需求文档内容

#### 3.4.3 OpenClaw集成说明
- **会话能力由OpenClaw提供**：前端调用OpenClaw Gateway API，传递用户输入，获取AI响应
- **OpenClaw职责**：
  - 按预设模板收集需求字段
  - 计算进度百分比（0-100%）
  - 生成结构化需求文档（Markdown/JSON）
  - 生成PPT（调用PPT生成Skill）
  - 生成原型（调用UI Agent生成HTML）
- **前端职责**：
  - 自定义设计样式（气泡、输入框、进度条等）
  - 渲染OpenClaw返回的内容（Markdown渲染、PDF预览、iframe嵌入等）

#### 3.4.4 需求文档预览与操作
- 右侧预览区顶部工具栏：
  - 版本选择下拉框（V1.0、V2.0等）
  - 三个标签：**需求文档**（默认）、**PPT**、**原型**
- 用户可滚动查看文档内容，预览区支持富文本渲染

#### 3.4.5 确认与生成其他文档
- 预览区右侧操作按钮组：
  - **确认需求**：将当前版本标记为已确认
  - **生成原型**：确认后可用，OpenClaw调用UI Agent生成可交互HTML原型
  - **生成PPT**：确认后可用，OpenClaw调用Skill生成PPT文件
  - **需求管理**：跳转到需求管理列表
  - **下载**：下载当前产出物

#### 3.4.6 需求调整与同步
- 用户可在对话区继续修改需求（OpenClaw会记录变更）
- 每次修改自动生成新版本（V2.0、V3.0等）
- 新版本需要重新确认后才能生成原型和PPT
- OpenClaw自动同步更新三份产出物（文档、PPT、原型）

### 3.5 需求管理后台
#### 3.5.1 入口
- 登录后，点击右上角头像 → "**我的需求**"
- 侧边栏导航：需求管理、概览（管理员）

#### 3.5.2 需求列表页
- 左侧边栏导航
- 顶部状态标签页：全部、待审核、开发中、已完成
- 搜索和筛选区域：
  - 搜索需求名称
  - 按客户筛选
  - 按状态筛选
- 表格列：
  - 需求名称
  - 客户
  - 提交人
  - 状态（草稿、已确认、已提交开发、开发中、已完成、已拒绝）
  - 项目文档数量
  - 更新时间
  - 创建时间
  - 操作（根据状态显示不同操作按钮）
- 支持按更新时间排序

#### 3.5.3 操作按钮逻辑
根据不同状态显示不同操作：

| 状态 | 可用操作 |
|------|----------|
| **草稿** | 编辑、删除 |
| **已确认/已拒绝** | 编辑、提交开发、下载 |
| **已提交开发** | 采纳、拒绝（管理员） |
| **开发中** | 强制完成、修改状态（管理员） |
| **已完成** | 下载、删除 |

#### 3.5.4 需求详情页
左侧边栏导航：
- 概览
- 需求文档
- 需求原型
- 汇报方案（PPT）
- 交付资料（已完成状态）
- 原始资料
- 历史对话

**概览页内容**：
- 需求基本信息和状态徽章
- 操作按钮组（根据状态动态变化）
- 项目进度时间轴
- 统计卡片（对话轮次、需求文档、汇报方案、需求原型、原始资料数量）
- 需求概览（背景、目标）
- 需求更新记录

**各预览页**：
- 需求文档：Markdown渲染展示
- 需求原型：iframe嵌入展示
- 汇报方案：PPT预览或下载
- 交付资料：安装包、部署文档列表
- 原始资料：附件列表（会议纪要、官网截图、VI规范等）
- 历史对话：会话历史列表，可点击进入继续对话

### 3.6 需求提交与开发协作
#### 3.6.1 提交开发
- 用户在需求详情页点击"提交开发"，需求状态变为"已提交开发"
- 管理员在需求管理列表看到"待审核"状态的需求
- 管理员可"采纳"或"拒绝"需求

#### 3.6.2 管理员采纳流程
管理员采纳需求后，系统自动执行以下操作：
1. 将需求状态更新为"开发中"
2. **将需求文档同步到Git仓库**
   - 创建Git分支或提交
   - 将需求文档（Markdown格式）推送至指定Git仓库
   - 记录Git提交信息到需求记录中
3. 通知交付团队开始开发

#### 3.6.3 Git同步功能
- **触发时机**：管理员点击"采纳"按钮后
- **同步内容**：
  - 需求文档（Markdown格式）
  - 需求元数据（JSON格式，包含需求ID、名称、状态等）
- **Git操作**：
  - 自动创建分支：`feature/requirement-{requirementId}`
  - 提交信息：`feat: add requirement #{requirementId} - {requirementName}`
  - 可选：创建Pull Request
- **配置项**：
  - Git仓库地址
  - 访问令牌/密钥
  - 目标分支（如main/master）
  - 文件存放路径模板

#### 3.6.4 状态流转
```
草稿 → 已确认 → 已提交开发 → 开发中 → 已完成
         ↑___________|__________↑
           （拒绝后可重新提交）
```

## 6. 技术实现建议

### 6.1 前端
- **框架**：React + TypeScript
- **构建工具**：Vite
- **路由**：React Router
- **状态管理**：Zustand
- **样式**：Tailwind CSS
- **图标**：Lucide React
- **UI组件**：自定义组件（Button、Card、Badge等）

### 6.2 后端
- **语言**：Node.js (Express/Nest) 或 Python (FastAPI/Django)
- **数据库**：PostgreSQL
- **文件存储**：OSS（阿里云/腾讯云）或本地存储
- **认证方式**：集成公司Ops系统SSO

### 6.3 OpenClaw集成
- **API接入**：调用OpenClaw Gateway API
- **接口说明**：
  - `POST /api/openclaw/chat`：发送用户输入，获取AI响应
  - `GET /api/openclaw/progress/{sessionId}`：获取当前收集进度
  - `POST /api/openclaw/generate/doc`：生成需求文档
  - `POST /api/openclaw/generate/ppt`：生成PPT
  - `POST /api/openclaw/generate/prototype`：生成原型
- **数据交互**：
  - 请求体包含：session_id、user_input、requirement_template
  - 响应包含：ai_message、progress、doc_content、artifacts
- **错误处理**：
  - OpenClaw服务异常时降级处理
  - 重试机制（最多3次）
  - 用户友好错误提示

### 6.4 Ops系统登录集成
- **SSO集成**：
  - 对接公司统一认证平台
  - 支持OAuth2.0/SAML协议
  - 登录后获取用户信息（姓名、部门、角色）
- **用户同步**：
  - 首次登录时同步用户信息到本地数据库
  - 定期同步用户组织架构变更
- **权限映射**：
  - Ops系统角色映射到平台角色（user/admin）
  - 支持按部门配置默认权限

### 6.5 Git同步功能
- **技术方案**：
  - 使用Git库（如Node.js的simple-git或Python的GitPython）
  - 或使用GitHub/GitLab API直接操作
- **实现步骤**：
  1. 配置Git仓库信息和访问凭证
  2. 管理员点击"采纳"时触发同步
  3. 克隆或拉取最新代码
  4. 在指定目录创建需求文档文件
  5. 提交并推送至远程仓库
  6. 记录Git信息到数据库
  7. 更新同步状态
- **错误处理**：
  - 同步失败时记录错误日志
  - 提供重试机制
  - 管理员可手动触发重新同步

## 7. 非功能性需求
- **安全性**：
  - 用户数据隔离，管理员权限校验
  - 文件上传需限制类型和大小
  - 集成公司Ops认证体系，确保账号安全
  - Git仓库访问凭证加密存储
- **性能**：
  - AI生成文档时间不超过30秒（如遇长任务可异步轮询）
  - OpenClaw API响应时间 < 2秒
  - 并发用户数初期支持50
- **可靠性**：
  - OpenClaw服务异常时降级处理
  - Git同步失败不影响主流程
  - 关键操作记录日志便于排查
- **可扩展性**：
  - 未来可对接更多AI模型
  - 支持更多产出格式
  - 模块化设计便于功能扩展
