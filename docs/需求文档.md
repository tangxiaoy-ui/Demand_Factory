# 需求工厂平台需求文档

## 1. 项目概述  
构建一个面向内部运营与需求管理的统一平台——“需求工厂”。平台提供需求收集、文档生成、多格式产出（PPT、原型）、需求管理、开发协作等全流程功能。运营人员可通过首页推广案例，用户通过AI引导完成需求收集，最终产出标准化需求文档，并可与开发团队协同交付。

---

## 2. 用户角色与权限

| 角色 | 权限描述 |
|------|----------|
| **普通用户**（已注册登录） | 可创建自己的需求、查看和管理自己的需求列表；可编辑、提交开发、下载产出物；可查看个人资料。 |
| **平台管理员** | 可查看所有用户的需求；可管理用户（增删改查）；可配置首页案例。 |
| **交付团队**（特殊角色或外部系统） | 可接收“已提交开发”的需求，更新需求状态、上传代码包和部署手册。 |

## 3. 功能模块详细说明

### 3.1 首页与案例展示  
- **页面组成**：  
  - 顶部导航栏：Logo、案例展示入口、登录/个人中心入口。  
  - **Banner区**：大图背景，文案介绍“需求工厂，智能生成需求文档与原型”，按钮“**立即体验**”。  
    - 点击“立即体验” → 新开浏览器Tab，跳转到 **AI引导需求收集页**（未登录用户需先登录/注册）。  
  - **案例展示区**：展示2个精选案例卡片。  
    - 每个卡片包含：案例标题、简介、标签、缩略图。  
    - 点击卡片 → 进入案例详情页（可查看完整案例文档、原型、PPT，仅供预览复用）。  
- **复用功能**：案例详情页提供“**复用此需求**”按钮，可将案例内容快速填入AI引导流程。

### 3.2 用户登录与个人中心  
#### 3.2.1 登录方式  
用户可通过以下两种方式登录：  
- **微信扫码登录**：点击“微信登录”按钮，弹出二维码，用户微信扫码后自动登录（需配置微信开放平台应用）。  
- **手机号登录**：切换至“手机号登录”标签页，输入手机号，获取短信验证码，输入验证码登录。  

#### 3.2.2 个人中心  
- 右上角头像下拉菜单：  
  - **我的需求**（进入需求管理）  
  - **个人设置**（可绑定/解绑手机号、微信）  
  - **退出登录**

### 3.3 AI引导需求收集与文档生成  

#### 3.3.1 页面布局  
页面布局根据需求收集状态动态变化：

**状态一：初始收集阶段（需求文档未生成）**  
- 页面为单列布局，仅显示**对话区**。  
- 对话区宽度为**400px～980px**，自适应居中。  
- 对话区顶部显示**需求收集进度百分比**（由OpenClaw计算并返回）。  
- 右侧无预览区（或显示提示“需求文档将在收集完成后展示”）。  
- **会话能力由OpenClaw提供**（负责对话理解、进度计算、最终文档生成），样式（如气泡、输入框、进度条）由本网站自定义设计。

**状态二：文档已生成后**  
- 页面切换为**两列分屏**：  
  - **左侧对话区**：宽度固定为400px（可交互），保留对话历史，用户可继续修改需求。  
  - **右侧预览区**：宽度占据剩余屏幕，用于展示需求文档、PPT预览、原型预览。  
- 右侧预览区顶部有标签页（需求文档、PPT、原型），支持切换。  
- 预览区内容由OpenClaw生成后返回，前端负责渲染（Markdown渲染、PDF预览、iframe嵌入等）。

#### 3.3.2 会话流程  
1. AI首先欢迎并询问需求基本信息（如项目名称、目标用户、核心功能、非功能需求、技术约束等）。  
2. 每轮对话，AI根据用户输入逐步完善需求结构。  
3. 进度百分比根据预设的**必填字段完成度**动态计算（由OpenClaw内部逻辑完成）。  
4. 当收集完成（100%）时，AI自动生成需求文档（结构化Markdown/JSON），并**自动切换为两列分屏**，右侧预览区显示生成的需求文档内容。

#### 3.3.3 需求文档预览与操作  
- 右侧预览区顶部工具栏：  
  - 三个标签：**需求文档**（默认）、**生成PPT**、**生成原型**。  
- 初始仅“需求文档”标签有效，其他标签需用户点击“**确认需求**”后才会生成。  
- 用户可滚动查看文档内容，预览区支持富文本渲染。

#### 3.3.4 确认与生成其他文档  
- 预览区顶部有按钮“**确认需求**”。  
  - 点击后：按钮文案变为“**生成PPT**”和“**生成原型**”（两个独立按钮）。  
  - 用户可分别点击生成PPT和原型。  
    - 生成PPT：调用后端服务生成PPT文件（如.pptx），并在预览区展示PDF预览或下载链接。  
    - 生成原型：调用UI Agent生成可交互HTML原型，并在预览区展示iframe预览。  
  - 生成后，顶部标签支持切换查看三份文档（需求文档、PPT预览、原型预览）。

#### 3.3.5 需求调整与同步  
- 用户可在对话区继续修改需求（AI会记录变更）。  
- 当需求文档内容被用户通过对话调整后，**三份产出物（文档、PPT、原型）自动同步更新**。  
- 同步机制：后台检测到需求文档内容变化后，重新生成PPT和原型，并更新预览。  
- 用户也可直接在预览区编辑需求文档（需实现富文本编辑器），保存后同样触发同步更新。

### 3.4 需求管理后台  
#### 3.4.1 入口  
- 登录后，点击右上角头像 → “**我的需求**”。

#### 3.4.2 需求列表页  
- 表格列：  
  - 需求名称  
  - 需求状态（草稿、已确认、已提交开发、开发中、已完成）  
  - 更新日期  
  - 操作（查看详情、编辑、提交开发、下载产出物）  
  - 创建人（管理员可见全部，普通用户只看到自己的）  
- 支持筛选、搜索。  
- 顶部按钮：**新建需求** → 跳转到AI引导页。

#### 3.4.3 需求详情页  
- 展示需求所有信息（需求文档内容、原始资料列表、历史版本、生成的文件列表）。  
- **文件列表**：  
  - 需求文档（.md/.html）  
  - PPT文件（.pptx）  
  - 原型HTML包（.zip）  
  - 代码包（.zip，由交付团队上传）  
  - 部署手册（.pdf/.md）  
- 用户可以上传原始资料（如草图、图片、文档）作为附件。  
- 操作按钮：  
  - 编辑需求（返回AI引导页或直接编辑文档）  
  - 提交开发（仅当需求状态为“已确认”时可用）  
  - 下载产出物（打包下载所有文件）

### 3.5 需求提交与开发协作  
#### 3.5.1 提交开发  
- 用户在需求详情页点击“提交开发”，需求状态变为“已提交开发”。  
- 系统将需求信息（包括文档、PPT、原型）推送到交付团队队列（可在另一系统“aicoing工厂”或本系统内实现）。  
- 交付团队可查看需求，更新状态（开发中、已完成），并上传代码包和部署手册。

#### 3.5.2 状态同步  
- 交付团队完成开发后，将状态更新为“已完成”，并上传代码包和部署手册。  
- 需求工厂自动拉取状态和文件，用户在需求详情页可下载。

## 5. 页面交互与UI要点

### 5.1 整体风格  
- 现代、简洁、科技感。  
- 主色调：蓝色（#3b82f6）搭配灰色背景。  
- 响应式布局，移动端适配（但重点为PC端）。

### 5.2 AI引导页布局细节  
- **收集阶段**：对话区居中，宽度自适应（400px~980px），背景留白。  
- **生成后**：左侧对话区固定宽400px，右侧预览区自动撑满。两列之间可拖拽调整宽度（可选）。  
- 移动端适配：当屏幕宽度小于768px时，自动切换为上下布局（对话区在上，预览区在下）。

### 5.3 预览区标签切换  
- 三个标签（需求文档、PPT、原型）采用类似浏览器标签页样式。  
- 点击不同标签，下方内容区域切换展示对应预览（需求文档用Markdown渲染，PPT用PDF.js预览，原型用iframe嵌入）。

### 5.4 需求管理列表  
- 卡片式或表格，支持搜索、排序（按更新时间）。  
- 状态用不同颜色的徽章展示。

### 5.5 需求详情页  
- 顶部显示需求名称、状态。  
- 左侧菜单：概览、文件列表、历史版本。  
- 右侧内容区域展示详情或文件列表。

## 6. 技术实现建议

### 6.1 前端  
- 框架：React/Vue + TypeScript。  
- UI组件库：Ant Design / Element Plus。  
- 富文本编辑：TipTap / Editor.js。  
- Markdown渲染：marked + highlight.js。  
- PDF预览：PDF.js。  
- 原型预览：iframe嵌入生成的HTML文件。  
- 微信扫码登录：使用官方 JS-SDK 或 OAuth2 流程，跳转授权后回调。  
- 手机号登录：集成第三方短信服务（阿里云/腾讯云）。

### 6.2 后端  
- 语言：Node.js (Express/Nest) 或 Python (FastAPI/Django)。  
- 数据库：PostgreSQL。  
- 文件存储：OSS（阿里云/腾讯云）或本地存储。  
- 认证方式：支持微信OAuth2、手机号+验证码（JWT token）。  
- 用户表设计支持多登录方式关联。  
- **AI集成**：会话能力完全由**OpenClaw**提供：前端调用OpenClaw Gateway API，传递用户输入，获取AI响应。  
  - OpenClaw需具备：按预设模板收集需求字段，计算进度；生成结构化需求文档（Markdown/JSON）；生成PPT和原型（调用相关Skill）。  
- 生成PPT：可使用 python-pptx 或 LibreOffice 命令行，或调用第三方API。  
- 生成原型：调用UI Agent（配置Tailwind + 交互代码）生成HTML，并保存为文件。

### 6.3 开发协作  
- 可采用Webhook或消息队列通知交付团队。  
- 交付团队可通过API更新需求状态和上传文件。

## 7. 非功能性需求  
- **安全性**：用户数据隔离，管理员权限校验；文件上传需限制类型和大小。  
- **性能**：AI生成文档时间不超过30秒（如遇长任务可异步轮询）；并发用户数初期支持50。  
- **可扩展性**：未来可对接更多AI模型、支持更多产出格式。  
- **日志与监控**：记录关键操作，便于排查问题。
