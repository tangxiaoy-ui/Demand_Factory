# 需求工厂平台 - 需求列表状态与操作说明

> 本文档定义需求列表页的状态流转、操作按钮逻辑及不同角色的权限范围。

---

## 一、需求状态定义

| 状态 | 说明 |
|------|------|
| **草稿** | 需求创建后初始状态，可编辑、删除 |
| **已确认** | 需求文档已生成并确认，可提交开发 |
| **已提交开发** | 已推送至管理员审核，等待采纳或拒绝 |
| **开发中** | 管理员已采纳并同步到Git仓库，开发团队开发中，仅可查看 |
| **已拒绝** | 管理员拒绝，可编辑后再次提交开发 |
| **已完成** | 开发完成，可下载最终交付物 |

---

## 二、普通用户（个人）需求列表

**展示范围**：当前登录用户自己创建的需求。

**列表列**：需求名称、需求状态、更新日期、操作列。

### 状态与操作对照表

| 状态 | 可用操作按钮 | 功能说明 |
|------|--------------|----------|
| **草稿** | 查看详情、编辑、删除 | 草稿状态可随时编辑或删除；编辑跳转至AI引导页继续收集。 |
| **已确认** | 查看详情、编辑、提交开发、下载产出物 | 提交开发后状态变更为"已提交开发"；下载产出物可打包下载需求文档、PPT、原型。 |
| **已提交开发** | 查看详情、撤回提交 | 等待管理员审核期间可撤回；撤回后状态回退至"已确认"。 |
| **开发中** | 查看详情 | 开发中不可修改或提交，仅可查看详情。Git同步完成后可查看Git信息。 |
| **已拒绝** | 查看详情、编辑、再次提交开发、下载产出物 | 查看拒绝原因，编辑修改后可再次提交开发；操作同"已确认"。 |
| **已完成** | 查看详情、下载产出物 | 可下载最终代码包和部署手册。 |

## 三、平台管理员需求列表

**展示范围**：所有用户的需求（含自己）。

**列表列**：需求名称、需求状态、更新日期、创建人、操作列。

### 状态与操作对照表

| 状态 | 可用操作按钮 | 功能说明 |
|------|--------------|----------|
| **草稿** | 查看详情、编辑、删除 | 管理员可编辑任何人草稿，也可删除。 |
| **已确认** | 查看详情、编辑、提交开发、下载产出物、强制归档 | 管理员可直接提交开发或强制归档（绕过流程）。 |
| **已提交开发** | 查看详情、采纳、拒绝 | **采纳**：触发Git同步，状态变更为"开发中"；**拒绝**：状态变更为"已拒绝"，需填写拒绝原因。 |
| **开发中** | 查看详情、强制完成、修改状态 | 可跳过开发流程直接标记为"已完成"，并上传交付物。可查看Git同步状态。 |
| **已拒绝** | 查看详情、编辑、采纳、删除 | 管理员可直接采纳（进入开发中并同步Git），或让用户修改后重新提交。 |
| **已完成** | 查看详情、下载产出物、删除 | 管理员可删除已完成需求，或重新打开（状态回退）。 |

### 管理员扩展权限

- 可在详情页为任何需求补充文件、修改需求内容、调整状态
- 可查看所有用户的"创建人"信息
- 可操作所有状态的需求（不受普通用户限制）
- 对"已提交开发"状态拥有审核权限（采纳/拒绝）
- **可触发Git同步**（点击采纳时自动触发）
- **可查看Git同步状态**和相关信息

## 四、Git同步功能说明（新增）

### 4.1 触发条件
- **自动触发**：管理员点击"采纳"按钮后
- **手动触发**：管理员可在需求详情页手动触发重新同步

### 4.2 同步流程
```
管理员点击"采纳"
       │
       ▼
系统创建Git分支：feature/requirement-{id}
       │
       ▼
生成Markdown文档文件
       │
       ▼
提交到Git仓库
       │
       ▼
记录Git信息到数据库
       │
       ▼
更新需求状态为"开发中"
```

### 4.3 同步状态

| 状态 | 说明 |
|------|------|
| **pending** | 等待同步 |
| **syncing** | 同步进行中 |
| **synced** | 同步成功 |
| **failed** | 同步失败 |

### 4.4 Git信息展示

在"开发中"和"已完成"状态的需求详情页，展示Git同步信息：

```
┌─────────────────────────────────────┐
│ Git同步状态                          │
├─────────────────────────────────────┤
│ 状态：✓ 同步成功                     │
│ 分支：feature/requirement-abc123     │
│ 提交：a1b2c3d                        │
│ 时间：2024-03-15 14:32:00            │
│ 操作：[重新同步] [查看仓库]          │
└─────────────────────────────────────┘
```

### 4.5 错误处理

- **同步失败时**：
  - 状态标记为"failed"
  - 记录错误日志
  - 显示错误原因（如认证失败、网络错误等）
  - 管理员可手动重试
  
- **失败原因**：
  - Git认证失败（检查访问令牌）
  - 网络连接超时
  - 仓库不存在或权限不足
  - 文件冲突

### 4.6 权限控制

- 只有管理员可以触发Git同步
- 普通用户只能查看Git同步状态，不能操作
- 同步失败不影响需求状态的流转

## 五、操作按钮详细说明

| 按钮 | 图标 | 触发条件 | 功能描述 |
|------|------|----------|----------|
| **查看详情** | 👁️ | 所有状态 | 跳转至需求详情页，查看完整信息、文件列表、历史版本、Git同步状态。 |
| **编辑** | ✏️ | 草稿、已确认、已拒绝 | 跳转至AI引导页，可修改需求内容。 |
| **删除** | 🗑️ | 草稿（普通用户）、已完成（管理员） | 删除需求记录，关联文件一并移除。 |
| **提交开发** | 📤 | 已确认、已拒绝 | 将需求状态变更为"已提交开发"，推送至管理员审核队列。 |
| **撤回提交** | ↩️ | 已提交开发 | 撤销提交，状态回退至"已确认"。 |
| **采纳** | ✅ | 已提交开发（管理员）、已拒绝（管理员） | 管理员审核通过，触发Git同步，状态变更为"开发中"。 |
| **拒绝** | ❌ | 已提交开发（管理员） | 管理员审核拒绝，状态变更为"已拒绝"，需填写拒绝原因。 |
| **下载产出物** | 📥 | 已确认、已拒绝、已完成 | 打包下载当前所有产出物（文档、PPT、原型、代码包等）。 |
| **强制归档** | ✓ | 管理员 | 将任意状态需求直接标记为"已完成"。 |
| **重新同步** | 🔄 | 开发中、已完成（管理员） | 手动触发Git同步。 |
| **查看仓库** | 🔗 | 开发中、已完成 | 跳转至Git仓库查看提交详情。 |

## 八、实现要点

### 8.1 权限判断

```typescript
// 判断当前用户是否为管理员
const isAdmin = user?.role === 'admin'

// 判断是否为需求所有者
const isOwner = req.userId === user?.id

// 普通用户只能操作自己的需求
// 管理员可操作所有需求，且对"已提交开发"有审核权限
// 只有管理员可以触发Git同步
```

### 8.2 状态更新

```typescript
// 提交开发（已确认或已拒绝状态）
updateRequirement(id, { status: 'submitted' })

// 撤回提交
updateRequirement(id, { status: 'confirmed' })

// 管理员采纳（触发Git同步）
updateRequirement(id, { 
  status: 'developing',
  gitSyncStatus: 'pending'
})
// 然后触发Git同步任务

// 管理员拒绝
updateRequirement(id, { 
  status: 'rejected', 
  rejectionReason: '原因...' 
})

// 强制完成（管理员）
updateRequirement(id, { status: 'completed' })

// Git同步成功回调
updateRequirement(id, {
  gitCommitHash: 'a1b2c3d',
  gitBranch: 'feature/requirement-abc123',
  gitSyncStatus: 'synced',
  gitSyncedAt: new Date()
})

// Git同步失败回调
updateRequirement(id, {
  gitSyncStatus: 'failed',
  gitSyncError: '错误原因...'
})
```

### 8.3 Git同步服务实现

```typescript
// Git同步服务
class GitSyncService {
  async syncRequirement(requirement: Requirement) {
    try {
      // 1. 更新状态为syncing
      await this.updateStatus(requirement.id, 'syncing')
      
      // 2. 创建Git分支
      const branch = `feature/requirement-${requirement.id}`
      await this.createBranch(branch)
      
      // 3. 生成文档文件
      const files = await this.generateFiles(requirement)
      
      // 4. 提交到Git
      const commitHash = await this.commitToGit(branch, files)
      
      // 5. 更新成功状态
      await this.updateSuccess(requirement.id, {
        gitBranch: branch,
        gitCommitHash: commitHash,
        gitSyncStatus: 'synced'
      })
      
    } catch (error) {
      // 6. 更新失败状态
      await this.updateFailure(requirement.id, error.message)
      throw error
    }
  }
}
```

### 8.4 操作按钮渲染逻辑

```typescript
const renderActionButtons = (req: Requirement, isAdmin: boolean) => {
  const buttons = []
  
  // 所有状态都可查看
  buttons.push(<ViewButton />)
  
  switch (req.status) {
    case 'draft':
      buttons.push(<EditButton />, <DeleteButton />)
      break
    case 'confirmed':
      buttons.push(<EditButton />, <SubmitButton />, <DownloadButton />)
      break
    case 'submitted':
      if (isAdmin) {
        // 采纳按钮触发Git同步
        buttons.push(<ApproveButton onApprove={() => triggerGitSync(req.id)} />, <RejectButton />)
      } else {
        buttons.push(<WithdrawButton />)
      }
      break
    case 'rejected':
      buttons.push(<EditButton />, <SubmitButton />, <DownloadButton />)
      if (isAdmin) buttons.push(<ApproveButton />, <DeleteButton />)
      break
    case 'developing':
      // 显示Git同步状态
      buttons.push(<GitSyncStatus status={req.gitSyncStatus} />)
      if (isAdmin) {
        buttons.push(<ForceCompleteButton />)
        if (req.gitSyncStatus === 'failed') {
          buttons.push(<RetryGitSyncButton />)
        }
      }
      break
    case 'completed':
      buttons.push(<DownloadButton />)
      if (isAdmin) buttons.push(<DeleteButton />)
      break
  }
  
  return buttons
}
```
